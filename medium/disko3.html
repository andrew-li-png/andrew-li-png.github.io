DISKO 3

Disk Image Autopsy Compressed File

Goal: Find the hidden flag in a disk image. Spoiler: it wasn’t sitting in plain text — it was tucked away inside a compressed file.

Challenge Description

Screenshot from the picoCTF challenge page:

Challenge Overview

DISKO 3 felt like the sibling of DISKO 2: same idea (disk image forensics), but this time the flag wasn’t immediately obvious from a quick string search. The hint basically says it: “it’s not as plain as you think.”

Instead of doing heavy searching, I loaded the image into Autopsy and simply browsed the filesystem until something stood out — a file named flag.gz. If you see .gz, your brain should instantly say: “compressed.”

Solution

1) Unzip the downloaded disk image

The challenge provides the disk image compressed, so first step was just extracting it with gunzip. 

gunzip disko-3.dd.gz

After this, you should have a plain .dd file ready to load into Autopsy.

2) Load the .dd into Autopsy

I added the extracted disk image as a data source in Autopsy. Once it finished ingesting, I started browsing the directory tree like a normal filesystem.

3) Browse to flag.gz and read the contents

This was the nice part: no keyword searches, no carving marathon, no “which partition is it?” Just browsing through the contents was enough to spot flag.gz.

Clicking flag.gz in Autopsy immediately displayed the contents in the viewer, and the flag was sitting right there.

Flag

picoCTF{n3v3r_z1p_2_h1d3_654235e0}

Quick lesson: if a file looks “too normal,” check for compression layers. .gz is basically a neon sign that says “open me.”

Tools Used

  • gunzip (extract the compressed disk image)
  • Autopsy (filesystem browsing + viewing .gz contents)
